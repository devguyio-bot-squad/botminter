# Demo Context: Interview Notes

Captured during the demo planning conversation on 2026-03-20.

## Target Audience

- People in the presenter's company and org
- All around software engineering — different functions, different forms
- Engineers, content writers, QE, some leadership in engineering
- All more or less engineers
- Deeply technical leadership
- They are all potential users, early adopters
- For leadership: potential adopters of BotMinter
- Org: OpenShift
- Project: HyperShift — the majority of the audience either knows it by heart, has used it, has heard of it, or works in an adjacent team

## Primary Goals

1. **Drive adoption** — show engineers that this would actually fit their needs
2. **Explain the vision** — achieve two sub-goals:
   - Show engineers that this fits their needs (presenter knows it does)
   - Establish the presenter as a subject matter expert in this area for any future agentic software development lifecycle discussions

## Distribution

- Ideally publish on YouTube and LinkedIn
- Will first share on Slack (internal)

## Format

- **Recorded** (not live)
- 10 minutes

## Demo Requirements

- Must be based on everything actually working end-to-end
- If something doesn't work, it will be fixed first before recording
- Real project: HyperShift
- Open to time-lapse vs. real-time — some things can be pre-recorded and shown as results

## Bridge / Chat

- Definitely wants to show the bridge
- At minimum: show agents joining and that there is a room
- Ideally also show agents sending messages there
- Depends on how easy it is to get working
- Concern: the current compact profile may need tweaks for human interaction instructions — they were in a separate compact-telegram profile that was deleted, unclear if merged back properly

## Biggest Pain Point (Audience)

- "We would like to go agentic software development lifecycle, but we don't know what the hell is that or how the hell can that work"
- "We have been using Claude Code and we don't know that there is a better way to do that"
- "There's a lot of stuff out there and we don't know"
- There is a perception that the current workflow today is not really something that will stay around — they are starting to explore the next workflow

## What the Presenter Struggled With

- Question 7 (single takeaway): "I feel that I want to show everything. There are different aspects matching different needs."
- Question 8 (demos liked/disliked): No answer — hasn't seen comparable demos
- Question 3 (how technical): "I don't know, that's why I'm asking" — but noted that leadership is deeply technical

## Key Insight from Interview

The takeaway should NOT be "BotMinter encodes your existing process" — the audience knows their current process needs to evolve. The message should be: **"This is what agentic SDLC actually looks like, and here's how to get there."**
