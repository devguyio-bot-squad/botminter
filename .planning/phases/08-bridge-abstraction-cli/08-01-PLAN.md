Create `crates/bm/src/bridge.rs` with:

**Structs (per CONTEXT.md locked decisions):**
- `BridgeManifest` — parsed from bridge.yml using serde_yml. Fields: api_version (renamed from apiVersion), kind, metadata (BridgeMetadata), spec (BridgeSpec)
- `BridgeMetadata` — name, display_name (Option), description (Option)
- `BridgeSpec` — bridge_type (renamed from "type"), config_schema (renamed from configSchema), lifecycle (Option<BridgeLifecycle>), identity (BridgeIdentitySpec), config_dir (renamed from configDir), room (Option<BridgeRoomSpec>)
- `BridgeLifecycle` — start, stop, health (all String recipe names)
- `BridgeIdentitySpec` — onboard, rotate_credentials (renamed from rotate-credentials), remove
- `BridgeRoomSpec` — create, list (String recipe names)
- `BridgeState` — bridge_name (Option), bridge_type (Option), service_url (Option), container_ids (Vec<String>), status (String), started_at (Option), last_health_check (Option), identities (HashMap<String, BridgeIdentity>), rooms (Vec<BridgeRoom>)
- `BridgeIdentity` — username, user_id, token, created_at (all String)
- `BridgeRoom` — name, room_id (Option<String>), created_at (String)

**Functions:**
- `state_path(workzone: &Path, team_name: &str) -> PathBuf` — returns `workzone/team_name/bridge-state.json`
- `load_manifest(bridge_dir: &Path) -> Result<BridgeManifest>` — reads and parses bridge.yml from bridge_dir
- `load_state(path: &Path) -> Result<BridgeState>` — returns default BridgeState if file missing, otherwise parses JSON
- `save_state(path: &Path, state: &BridgeState) -> Result<()>` — atomic write with 0600 permissions (same pattern as topology.rs: write to temp file, set permissions, rename)
- `discover(team_repo: &Path, team_name: &str) -> Result<Option<PathBuf>>` — reads botminter.yml, checks for `bridge` key, returns path to `bridges/{name}/` directory. Returns Ok(None) if no bridge key. Bails with actionable error if bridge dir doesn't exist.
- `invoke_recipe(bridge_dir: &Path, recipe: &str, args: &[&str], team_name: &str) -> Result<Option<serde_json::Value>>` — creates tempdir for BRIDGE_CONFIG_DIR, runs `just --justfile {bridge_dir}/Justfile {recipe} {args}` with env vars BRIDGE_CONFIG_DIR and BM_TEAM_NAME, reads config.json from tempdir if it exists. Sets working directory to bridge_dir. Returns Ok(None) if no config.json written.
- `resolve_credential(identity_name: &str, state: &BridgeState) -> Option<String>` — checks env var `BM_BRIDGE_TOKEN_{USERNAME}` (uppercased) first, then state file identity token
