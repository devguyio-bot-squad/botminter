  <files>.planning/spikes/rc-podman-spike/spike.sh</files>
  <action>
Create a standalone bash script at `.planning/spikes/rc-podman-spike/spike.sh` that:

1. **Creates a Podman Pod** named `bm-rc-spike` with port mapping (pick a free port, default 3100 to avoid conflicts):
   - `podman pod create --name bm-rc-spike -p 3100:3000`

2. **Starts MongoDB** in the pod with single-node replica set:
   - `podman run -d --pod bm-rc-spike --name bm-rc-spike-mongo docker.io/mongo:7.0 mongod --replSet rs0 --oplogSize 128`
   - Wait for MongoDB to accept connections (poll with `podman exec ... mongosh --eval "db.runCommand({ping:1})"`)
   - Initialize replica set: `podman exec ... mongosh --eval 'rs.initiate({_id:"rs0",members:[{_id:0,host:"localhost:27017"}]})'`

3. **Starts Rocket.Chat** in the pod:
   - Use `registry.rocket.chat/rocketchat/rocket.chat:latest` image
   - Set env vars: `ROOT_URL=http://localhost:3100`, `MONGO_URL=mongodb://localhost:27017/rocketchat?replicaSet=rs0`, `MONGO_OPLOG_URL=mongodb://localhost:27017/local?replicaSet=rs0`, `OVERWRITE_SETTING_Show_Setup_Wizard=completed`, `ADMIN_USERNAME=rcadmin`, `ADMIN_PASS=rcadmin123`, `ADMIN_EMAIL=admin@botminter.local`

4. **Health check loop** -- poll `http://localhost:3100/api/v1/info` with backoff (max 120 seconds, RC takes 30-60s to boot)

5. **Admin login** -- `POST /api/v1/login` with admin credentials, extract `authToken` and `userId` via `jq`

6. **Create bot user** -- `POST /api/v1/users.create` with `username=spike-bot`, `roles=["bot"]`, random password via `openssl rand -hex 16`

7. **Generate bot auth token** -- `POST /api/v1/users.createToken` for the bot user, extract the token

8. **Create a channel** -- `POST /api/v1/channels.create` with name `spike-test`

9. **Verify REST API connectivity** -- `GET /api/v1/channels.list` with bot credentials, confirm the channel exists

10. **Bidirectional messaging proof via curl** -- send a message as the bot user to the `spike-test` channel via `POST /api/v1/chat.sendMessage`, then send a reply as the admin user, then retrieve messages via `GET /api/v1/channels.history` with bot credentials and confirm both messages are visible. Also send a bot command message (`/status`) and retrieve it to prove command-shaped messages transit correctly (RC-06 transport layer proof).

11. **Cleanup** -- `podman pod stop bm-rc-spike && podman pod rm bm-rc-spike`

The script should:
- Use `set -euo pipefail`
- Print clear progress messages to stderr
- Have a `cleanup()` function registered with `trap ... EXIT`
- Accept an optional `--keep` flag to skip cleanup for manual inspection
- Output a JSON summary at the end with: `admin_user_id`, `bot_user_id`, `bot_auth_token`, `channel_id`, `server_url`, `messages_sent` (count), `bidirectional_test` (pass/fail)
