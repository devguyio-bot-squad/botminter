# Requirements: BotMinter — Minty and Friends

**Defined:** 2026-03-04
**Core Value:** Operators can stand up and manage autonomous agentic teams that coordinate through GitHub

## v0.06 Requirements

Requirements for the v0.06 Minty and Friends milestone. Derived from `specs/milestones/minty-and-friends-rapid/requirements.md` (19 Q&A entries) and `specs/milestones/minty-and-friends-rapid/design.md`.

### Coding-Agent-Agnostic

- [x] **CAA-01**: Agent tag filter library processes `+agent:NAME/-agent` tags in files with HTML and Hash comment syntax
- [x] **CAA-02**: `CodingAgentDef` data model with profile-level default and team-level override resolution
- [x] **CAA-03**: Profile restructuring — `coding-agent/` directory replaces `agent/`, `context.md` with inline tags replaces `CLAUDE.md`
- [x] **CAA-04**: Profile and member extraction transforms unified files to agent-specific — filters `+agent:NAME` tags and renames `context.md` to resolved `context_file` (e.g., `CLAUDE.md`)
- [x] **CAA-05**: Workspace parameterization — all hardcoded "claude-code" strings replaced with resolved agent config
- [x] **CAA-06**: Sprint 1 documentation updated in `docs/content/`

### Profile Externalization

- [x] **PROF-01**: `bm profiles init` command extracts embedded profiles to `~/.config/botminter/profiles/`
- [x] **PROF-02**: Overwrite control with warning prompts and `--force` flag for scripted use
- [x] **PROF-03**: Disk-based profile API — all profile operations read from disk instead of embedded data
- [x] **PROF-04**: Auto-prompt pattern — commands detect missing disk profiles and offer inline initialization
- [x] **PROF-05**: Sprint 2 documentation updated

### Workspace Repository

- [x] **WRKS-01**: Workspace repo creation — `bm teams sync --push` creates GitHub-hosted workspace repos with submodules
- [x] **WRKS-02**: Context assembly — `CLAUDE.md`, `ralph.yml`, `PROMPT.md` surfaced at workspace repo root
- [x] **WRKS-03**: Workspace sync for existing repos — `bm teams sync` updates existing workspace repos
- [x] **WRKS-04**: `bm start/stop` adapted for new workspace repo model
- [x] **WRKS-05**: Status commands show workspace repo state and health
- [x] **WRKS-06**: Sprint 3 documentation updated

### Skills Extraction

- [x] **SKIL-01**: Board-scanner skill migrated from embedded hat instructions to composable skill format
- [x] **SKIL-02**: Profile directory restructured — `members/` renamed to `roles/`, stale `.botminter/` path references updated
- [x] **SKIL-03**: Ralph prompt shipping — embedded hat/process logic extracted to `ralph-prompts/` directory
- [x] **SKIL-04**: Status-workflow skill — composable skill for status transitions and issue operations
- [x] **SKIL-05**: Sprint 4 documentation updated

### Team Manager

- [x] **TMGR-01**: Team Manager role definition in profile manifest with minimal statuses (`mgr:todo/in-progress/done`)
- [x] **TMGR-02**: Team Manager skeleton — member content files (ralph.yml, PROMPT.md, context.md, knowledge)
- [x] **TMGR-03**: Team Manager's default project is the team repo itself

### Interactive Sessions

- [x] **CHAT-01**: `bm chat <member>` launches coding agent session in member's workspace repo
- [x] **CHAT-02**: Meta-prompt function builds context-aware system prompt from member workspace
- [x] **CHAT-03**: Sprint 5 documentation updated

### Minty

- [x] **MNTY-01**: `bm minty` command launches coding agent session in BotMinter context
- [x] **MNTY-02**: Minty config structure at `~/.config/botminter/minty/` with system prompt and skill registry
- [x] **MNTY-03**: Minty skills — 4 composable capabilities (team-overview, profile-browser, hire-guide, workspace-doctor)
- [x] **MNTY-04**: Sprint 6 documentation updated

## Out of Scope

| Feature | Reason |
|---------|--------|
| Multi-coding-agent support (Gemini CLI, etc.) | Architecture is pluggable but only Claude Code implemented (A5) |
| Migration paths for existing teams | Alpha policy — every change is breaking (A16) |
| Selective profile extraction | Extract all at once, user deletes unwanted (A8) |
