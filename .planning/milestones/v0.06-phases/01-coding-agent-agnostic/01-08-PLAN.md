**Fix A — daemon_stop_terminates_running_members (around line 304):**

The current code sleeps 4s then checks `if stub_pid_file.exists()`. The stub PID file only appears once the daemon launches a member, which is race-dependent.

Replace the static sleep + conditional with a polling loop that waits for the stub PID file to appear (with timeout), then unconditionally asserts child termination:

```rust
// Wait for the daemon to launch a member (stub writes .ralph-stub-pid on start)
let workspace = tmp
    .path()
    .join("workspaces")
    .join(&team_name)
    .join(&member)
    .join("workspace");
let stub_pid_file = workspace.join(".ralph-stub-pid");

// Poll for stub PID file with timeout (daemon poll interval is 2s, so 15s is generous)
let poll_deadline = std::time::Instant::now() + Duration::from_secs(15);
while !stub_pid_file.exists() {
    assert!(
        std::time::Instant::now() < poll_deadline,
        "Timed out waiting for daemon to launch member — stub PID file never appeared at {}",
        stub_pid_file.display()
    );
    std::thread::sleep(Duration::from_millis(250));
}

// Read stub PID before stopping daemon
let stub_pid: u32 = fs::read_to_string(&stub_pid_file)
    .unwrap()
    .trim()
    .parse()
    .unwrap();
assert!(is_alive(stub_pid), "Stub ralph PID {} should be alive before daemon stop", stub_pid);

// Stop daemon
assert_cmd_success(&mut daemon_cmd(
    tmp.path(),
    &stub_dir,
    &["daemon", "stop", "-t", &team_name],
));

// Daemon should be dead
wait_for_exit(daemon_pid, Duration::from_secs(10));
assert!(
    !is_alive(daemon_pid),
    "Daemon PID {} should be dead after stop",
    daemon_pid
);

// Child member should also be dead
wait_for_exit(stub_pid, Duration::from_secs(10));
assert!(
    !is_alive(stub_pid),
    "Stub ralph PID {} should be dead after daemon stop",
    stub_pid
);
```

Move the workspace/stub_pid_file setup ABOVE the daemon stop command (currently it is below). The stub PID polling and assertion must happen between daemon start and daemon stop.

The overall flow should be: start daemon -> poll for stub PID file -> read stub PID and assert alive -> stop daemon -> assert daemon dead -> assert stub dead.

**Fix B — daemon_per_member_log_created (around line 409):**

The double-conditional block with decorative eprintln is useless. Replace it with actual assertions.

Two options based on what the daemon actually does:
1. If the daemon creates per-member log files at a known path, assert that path exists.
2. If per-member log creation requires successful member launch (which needs gh auth), the test cannot reliably verify per-member logs in all environments.

Given the test environment likely lacks gh auth, take the pragmatic approach: rename the test to `daemon_log_created` (matching what it actually verifies) AND add an assertion on daemon log content mentioning member launch attempt. Replace the decorative double-conditional with:

```rust
// Daemon log must exist
assert!(
    daemon_log.exists(),
    "Daemon log file should exist at {}",
    daemon_log.display()
);

let log_content = fs::read_to_string(&daemon_log).unwrap();
// Daemon should have attempted to process members (even if launch fails due to gh auth)
// Check that the log is not empty — daemon wrote something during its poll cycles
assert!(
    !log_content.is_empty(),
    "Daemon log should not be empty — daemon ran for 5 seconds with poll interval 2s"
);

// Check for per-member log file if a member was actually launched
let member_log = tmp
    .path()
    .join(format!(".botminter/logs/member-{}-{}.log", team_name, member));
if member_log.exists() {
    let member_log_content = fs::read_to_string(&member_log).unwrap();
    assert!(
        !member_log_content.is_empty(),
        "Per-member log should not be empty if created"
    );
}
// Note: per-member log creation requires successful member launch which needs gh auth.
// In CI/test environments without gh, the daemon log confirms the daemon ran but member
// launch may fail. The conditional on member_log is acceptable here because member launch
// is environment-dependent, unlike state.json which production code always writes.
```

Rename the test function from `daemon_per_member_log_created` to `daemon_log_created_on_poll` to accurately reflect what it verifies in all environments.
