SKILL.md frontmatter format (from profiles):
```yaml
---
name: gh
description: Manages GitHub Projects v2 workflows...
---
```
Some SKILL.md files use `description: >-` multiline syntax. The `name` and `description` fields are always present in YAML frontmatter.

ralph.yml skills section format:
```yaml
skills:
  enabled: true
  dirs:
    - team/coding-agent/skills
    - team/projects/<project>/coding-agent/skills
    - team/members/superman/coding-agent/skills
```

Paths in `skills.dirs` are relative to the workspace root. Each dir contains subdirectories, each with a SKILL.md file (e.g., `team/coding-agent/skills/gh/SKILL.md`).

Target output format in meta-prompt:
```markdown
## Skills

Available skills you can load on demand:

| Skill | Description | Load Command |
|-------|-------------|--------------|
| gh | Manages GitHub Projects v2 workflows... | Read `team/coding-agent/skills/gh/SKILL.md` |
| status-workflow | Performs status transitions... | Read `team/coding-agent/skills/status-workflow/SKILL.md` |

To use a skill, read its SKILL.md file for full instructions.
```
